# Issue Tracker: Repository Work Items

Engineering work for this repository is tracked in versioned Markdown files under `docs/work-items/`.

## Conventions

- The index is `docs/work-items/index.md`.
- Each bounded change is `docs/work-items/WI-<number>-<slug>.md`.
- A work item records its objective, context, scope, constraints, acceptance criteria, verification, and completion evidence.
- Status is recorded near the top and mirrored in the index.
- Product intent lives in `docs/product-spec.md`; architectural decisions live in `docs/decisions/`.

## Skill mapping

When a skill says to publish an issue, create the next numbered work item and update the index. When a skill says to fetch an issue, read the referenced work-item file and its linked specifications or decisions. Do not create a parallel `.scratch/` tracker.
