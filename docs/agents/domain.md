# Domain Documentation

The repository has one shared Aurea commerce context across its Next.js storefront and standalone Sanity Studio.

Before engineering work, read the relevant parts of:

- `docs/product-spec.md` for product intent and business rules.
- `docs/architecture.md` for system ownership and data flow.
- `docs/current-state.md` for implemented capabilities and known risks.
- `docs/decisions/` for accepted architectural decisions.
- The assigned file in `docs/work-items/` for bounded scope and acceptance evidence.

Use the established terms Product, Cart Item, Checkout Payload, Order, Order Item, Payment Verification, Sanity catalog, Supabase transaction store, and Paystack provider consistently. If a proposal contradicts an accepted decision, surface the conflict instead of silently changing it.
